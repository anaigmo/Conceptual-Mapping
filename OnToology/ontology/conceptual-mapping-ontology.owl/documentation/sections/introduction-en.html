Introduction back to ToC

The Conceptual Mapping ontology defines a model that abstracts the features that are present in state-of-the-art mapping languages, while considering their heterogeneity. The context of this model encompasses Ontology Based Data Access (OBDA) and Ontology Based Data Integration (OBDI), where data from several heterogeneous sources need to be accessible according to a unified and global view. Mappings are used to describe the relationships among the global view and the data source schemas. The diversity of available mapping languages motivates the construction of a language that gathers their features to, in the end, be able to represent them. This can potentially enhance the language interoperability and translation.
